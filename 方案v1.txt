# 加密货币量化交易执行纪律系统（完整实施方案）

---

## 一、项目目标

构建一套**可长期稳定运行的量化交易系统**，重点不在“预测市场”，而在：

* 强制执行策略
* 自动控制风险
* 防止人为干预
* 实现长期稳定收益

---

## 二、系统总体架构

```
数据层 → 特征工程 → 信号模型 → 仓位管理 → 风控系统 → 执行系统 → 日志系统
```

### 模块说明：

| 模块   | 作用       |
| ---- | -------- |
| 数据层  | 获取行情数据   |
| 信号模型 | 生成交易信号   |
| 仓位管理 | 控制每笔交易大小 |
| 风控系统 | 防止爆仓与大回撤 |
| 执行系统 | 自动下单     |
| 日志系统 | 记录所有行为   |

---

## 三、执行纪律规则（核心）

### 1. 交易规则

* 禁止手动交易
* 所有交易必须来自系统信号
* 禁止临时修改参数
* 禁止“跳过信号”

---

### 2. 仓位管理

* 单笔风险 ≤ 2%
* 总仓位 ≤ 100%
* 单资产 ≤ 30%

---

### 3. 风控规则

| 回撤   | 动作        |
| ---- | --------- |
| ≥10% | 仓位减半      |
| ≥15% | 停止开仓      |
| ≥20% | 清仓 + 停止系统 |

---

### 4. 交易频率限制

* 每小时最多一次调仓
* 信号变化小 → 不交易

---

## 四、策略模型（基础版本）

### 因子：

* 趋势（EMA）
* 均值回归（Z-score）
* 动量（Momentum）

### 信号计算：

```
signal = 0.4 * trend + 0.3 * mean + 0.3 * momentum
```

### 决策逻辑：

```
signal > 0.5 → 做多  
signal < -0.5 → 做空  
否则 → 空仓
```

---

## 五、风控系统设计

### 1. 回撤计算

```
drawdown = (当前资金 - 历史最高资金) / 历史最高资金
```

---

### 2. 风控逻辑

```
if drawdown < -20%:
    清仓 + 停止交易
elif drawdown < -15%:
    禁止开仓
elif drawdown < -10%:
    仓位减半
```

---

### 3. 仓位计算

```
position = signal × (账户资金 × 2%) ÷ 当前价格
```

## 六、执行系统

### 执行流程：

```
接收信号
→ 计算仓位
→ 风控过滤
→ 下单执行
→ 记录日志
```

---

### 关键原则：

* 只在信号变化时交易
* 避免频繁调仓
* 控制滑点
