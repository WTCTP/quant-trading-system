## 二、架构演进全记录

### 阶段 0：原始设计（已废弃）
- **架构**：三因子加权（EMA趋势 + Z-score均值回归 + 动量）→ 信号 → 阈值决策
- **问题**：因子是手工设计的，没有统计学习，无法捕捉市场微观结构

### 阶段 1：ML分类 + 组合优化（v1）
- **架构**：Logistic Regression → 预测方向概率 → Σ⁻¹μ 组合优化
- **Alpha 模型**：7个行为特征（成交量异常、价格冲击、波动率状态、突破强度、趋势连续性、流动性压缩）
- **标签**：未来5根K线是否上涨（二分类）
- **问题**：信号无优先级，所有信号一视同仁
- **诊断工具**：
  - 实验1（DISABLE_TRADING）：验证 alpha 是否来自过滤系统
  - 实验2（SIGNAL_INVERT）：验证信号方向
  - 实验3（SIGNAL_CONFIRM_BARS）：延迟确认

### 阶段 2：Regime Switching（v1.5）
- **架构**：按波动率状态（vol_20/vol_100）拆分为 low/mid 两个子模型
- **Regime 定义**：低波(ratio<0.90) / 中波(0.90~1.10) / 高波(ratio>1.10)
- **关键发现**：高波动状态模型不交易（噪声太大）
- **新增**：全局波动率过滤（BTC vol_regime > 1.3 → 全部空仓）

### 阶段 3：信号分层 + 诊断深化（v1.6）
- **关键发现**：信号不是均匀有效的，alpha 集中在顶部 20%
- **新增分析**：
  - 信号强度分层（10档 pd.qcut）
  - 时间稳定性分析（按年份拆分）
  - Q8（分位数80%+）: Sharpe 10.66，Q9（90%+）: Sharpe 6.60
- **尝试**：分层仓位（Q8=1.0权重, Q9=0.8）→ 效果不佳
- **根本原因**：NO_TRADE_ZONE 把样本空间砍没了（只有0.7%的K线通过）

### 阶段 4：暴露管理范式（v2.0）
- **核心洞察**：alpha 是"路径型的"而非"点触发型的"
  - 中波期间的持有 (+4.39%) 才是真正的 alpha 来源
  - 点状入场出场会吃掉所有 alpha
- **范式转换**：从"是否交易" → "持多少仓"
  - 中波内始终持有基础仓位（30%）
  - 信号只控制仓位大小（30% → 60% → 100%），不决定是否在场
- **新问题**：3406笔交易 — 信号每根K线都在波动，暴露乘数跟着变，触发不断的调仓

### 阶段 5：迟滞层（v2.1）
- **暴露等级迟滞**：
  - 追踪当前暴露等级（base/mid/full）
  - **仅当目标等级与当前等级不同时**才允许调仓
  - 同等级内信号波动 → mid_hold，不执行任何操作
  - 结果：中波-持有 +4.98%，交易从3406降到~3500

- **Regime EMA 平滑**：
  - vol_regime 用 EMA(span=12) 平滑后再做状态分类
  - 解决单根K线波动导致的 regime 频繁闪烁
  - 进入确认(2bar) + 退出缓冲(6bar)

- **新问题**：交易数仍高（~3500），主要来自 regime 进出而非等级内调仓

### 阶段 6：时机精度优化（v2.1 最终版）—— 当前版本
- **进场偏移测试**：
  - 测试信号触发后延迟0~4根K线入场的收益差异
  - Q9信号：shift=0 → +2.72%, shift=2 → **+8.06%**（3倍差距！）
  - **结论**：系统早进了2根K线

- **升仓延迟确认**：
  - 向上调仓（base→mid→full）：等2根K线确认后再执行
  - 延迟期间验证：价格突破（至少一次）+ 信号维持
  - 向下调仓（full→mid→base）：立即执行（风控优先）
  - **非对称设计**：升仓慢，降仓快

- **价格确认改"累积"**：
  - 错误方式：延迟的每根K线都要求BTC价格突破
  - 正确方式：延迟期间**至少有一次**突破即可
  - 突破后才意识到2520 bar卡在延迟中的原因

- **最终结果**：+5.14%, Sharpe 2.00, MaxDD -0.82%, 896笔交易

## 三、关键问题与解决方案

### 问题1：alpha 被过滤系统吃掉
- **现象**：中波-交易 -0.32% vs 中波-跳过 +1.89%
- **诊断**：DISABLE_TRADING + SIGNAL_INVERT 实验
- **根因**：信号方向正确，但 NO_TRADE_ZONE 太宽把世界砍没了
- **解决**：改用暴露管理（持续在场 + 信号调仓位）

### 问题2：信号分层坍塌
- **现象**：降低 NO_TRADE_ZONE 后用分层仓位 → Sharpe 直接崩塌
- **根因**：信号噪音太大，分层无法有效区分
- **Insight**：alpha 是路径型的，分位数分层是静态指标

### 问题3：暴露波动 → 交易爆炸
- **现象**：3406笔交易，费用吃掉所有利润
- **根因**：每根K线的信号波动 → 暴露乘数变化 → 调仓
- **解决**：暴露等级迟滞（仅在等级变化时调仓）

### 问题4：Regime 频繁闪烁
- **现象**：vol_regime 在阈值附近反复横跳
- **根因**：vol_20/vol_100 对单根K线敏感
- **解决**：EMA(12)平滑 + 进入确认 + 退出缓冲

### 问题5：进场太早
- **现象**：进场偏移测试显示 shift=2 收益是 shift=0 的3倍
- **根因**：信号触发后立即进场，但收益在2根K线后才兑现
- **解决**：升仓延迟2根K线 + 价格确认

### 问题6：价格确认条件过严
- **现象**：2520 bar 卡在延迟中，0笔交易
- **根因**：要求延迟的每根K线都出现BTC价格突破
- **解决**：改为"延迟期间至少一次突破"（累积检查）

## 四、核心设计原则（从血泪中提取）

1. **路径型 alpha，不要点触发** — 中波期间的持续暴露才是利润来源
2. **迟滞层必须存在** — 没有迟滞的定量系统是噪声放大器
3. **非对称执行** — 升仓慢（确认后进），降仓快（风控优先）
4. **价格确认要累积，不要逐根** — "至少一次" > "每根都要"
5. **先诊断，后治疗** — 禁用交易/信号反转/偏移测试 等诊断工具价值巨大
6. **不要过早优化信号** — 同样的信号，好的执行能让收益翻3倍

## 六、后续提升方向

### 优先级1：信号 → 仓位映射优化
- 当前：信号强度 → 离散三档（30%/60%/100%）
- 方向：连续映射函数（信号越强仓位越大），减少等级跳变
- 预期收益：+1~2%

### 优先级2：退出时机优化
- 当前：降仓立即执行（快但不一定最优）
- 方向：退出也加轻度延迟（2-3 bar），减少假突破导致的过早退出
- 证据：强制持有 +1.65%, Sharpe 10.35 → 退出经常是对的
- 预期收益：+0.5~1%

### 优先级3：多时间框架信号
- 当前：仅用1h K线的5-bar forward 标签
- 方向：加入4h级别的大趋势判断，避免逆大势交易
- 预期收益：不确定

### 优先级4：特征工程
- 当前：7个手工特征
- 方向：加入更高阶特征（波动率偏度、成交量剖面、跨币种相关性）
- 预期收益：+1~3%

### 优先级5：退出规则非对称化
- 当前：退出阈值 = 进入阈值
- 方向：exit_threshold < entry_threshold（让利润跑）
- 预期收益：+1~2%
